# Screenshot Guide

Use this folder for README screenshots.

## Required filenames
- `01-login.png`
- `02-sales-dashboard.png`
- `03-manager-dashboard.png`
- `04-director-dashboard.png`
- `05-procurement-duplicate-rule.png`
- `06-sales-credit-type-validation.png`
- `07-credit-payments.png`
- `08-mobile-view.png`

## Professional standards
- Format: `PNG`
- Width: `1600px` to `2200px` (desktop), `750px` to `900px` (mobile)
- Keep browser UI visible only if clean and intentional
- Use realistic but non-sensitive data
- Remove/blur secrets, phone numbers, personal IDs, and tokens
- Use consistent zoom level and similar spacing across screenshots
- Capture success/error toasts where relevant for workflows

## Capture checklist
- Login: clean branding and no clutter
- Sales: stock table + sale form + confirmation
- Manager: procurement/restock panels + updated stock
- Director: overview cards + users/branches table
- Duplicate validation: clear error message visible
- Type consistency validation: clear rejection message visible
- Credit payment: payment action and updated status visible
- Mobile: sidebar open/closed behavior and readable content
